# Learning Pattern - Ankita

## Core Pattern

- Fast conceptual understanding of algorithms and system design.
- Main bottleneck is concept-to-code translation speed under pressure.
- This is not a theory gap; it is an execution-sequencing gap under cognitive load.

## Failure Mode Under Pressure

- Blank-page coding plus ambiguity triggers working-memory overload.
- You tend to think in whole-solution form, then freeze at micro construction steps.
- Stress increases restlessness and slows implementation further.

## Strengths

- System thinking and architecture-level reasoning.
- Algorithm understanding and tradeoff analysis.
- Senior-level technical communication.

## Weaknesses (Fixable)

- Incremental code construction under time pressure.
- Holding multi-step implementation details in short-term memory.
- Starting from a blank editor without scaffolding.

## Coaching Approach That Works

- Forced stepwise scaffolding (Node -> constructor -> helpers -> API methods).
- Template reuse.
- Repeated rebuild of the same problem.
- Low-cognitive-load entry before full solution assembly.

## Coaching Approach To Avoid

- Jumping to harder/new problems before sequencing stabilizes.
- Additional theory-first study when execution sequencing is the blocker.

## Practical Diagnosis

You do not have a DSA knowledge problem.
You have a code-construction sequencing problem under cognitive load.

## Interview Framing

Conceptually senior, executionally overloaded under blank-page coding.

## Preferred Coaching Style

- Direct
- Structured
- Calm
- One step at a time with explicit checkpoints
